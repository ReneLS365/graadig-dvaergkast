# Game Design

## Core fantasy

Du kaster en dværg gennem en dødelig mine. Spilleren styrer stigning/fald, samler loot, banker score, overlever waves og bygger sin helt stærkere over tid.

Det skal føles som:

```txt
Jeg døde fordi jeg blev grådig.
Jeg kan komme længere med bedre build.
Jeg vil lige have nok skrot til ét våben-level mere.
```

## Modes

### Overlevelse

Primær mode. Waves bliver hårdere. Loot crates, våben, skrot, data og relics driver progressionen.

### Kampagne

Baner med mål, stigende sværhedsgrad og stjerner. Skal fungere som træning, onboarding og skill-test.

### Dagens Mine

Fair mode. Samme seed og standard-build for alle. Leaderboard uden pay-to-win.

### Duel

Samme seed og målscore. Social konkurrence.

## Progression

Progressionen skal være langsommere end score-eksplosionen. Score må være vild. XP, data og level skal være kontrolleret.

Derfor er v10/v11 bygget med dæmpet XP-skalering og højere krav til skills/våben.

## Fairness

```txt
Daily + Duel = standard-build
Survival + Campaign + Random + Chaos = dit build gælder
```

Det beskytter konkurrencen uden at dræbe progressionen.
